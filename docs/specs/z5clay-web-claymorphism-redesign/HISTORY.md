# History

## Initial Decision

The redesign was requested as a full-coverage UI refresh using a high-fidelity claymorphism design system. Because Tavily Hikari is an operational product, the adopted direction uses clay materiality globally while keeping admin data surfaces restrained and scannable.

## Dark Theme Repair

Dark mode was promoted from compatibility coverage to a first-class low-light tropical clay theme. The repair keeps the existing product information architecture and API contracts unchanged while replacing hard-coded blue-black/glass treatments, bright rim lights, and large purple glows with shared OKLCH-tinted neutrals, restrained semantic accents, and shorter clay shadows.
